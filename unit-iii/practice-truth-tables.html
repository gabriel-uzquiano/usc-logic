The following exercises cover the use of truth tables to check whether a formula is a tautology, whether two formulas are logically equivalent, and whether an argument is valid.

1. Simple truth tables

Populate the truth table for each formula. Sentence-letter columns are pre-filled. Click each ? cell to cycle through T and F, then click Check to verify your answers.

2. Tautologies

A formula is a tautology if and only if it is true under every assignment of truth values. Enter the formula in the Build card to construct the truth table. Then use the Check card and click Evaluate to determine whether it is a tautology. If it is not, a counterexample row will be highlighted.
2.1    ((p ∧ ¬q) → ¬(p → q))
Open ↗
2.2    (p → (q → p))
Open ↗
2.3    ((p ∨ q) → (p ∧ q))
Open ↗
2.4    ((p → (q → r)) → ((p → q) → r))
Open ↗

3. Logical equivalence

Two formulas are logically equivalent if they have the same truth value on every row. Use the Check card and click Evaluate to compare the two formulas. If they are not equivalent, the rows where their truth values differ will be highlighted.
3.1    ¬(p ∧ q)   and   (¬p ∨ q)
Open ↗
3.2    (p ∧ ¬q)   and   ¬(p → q)
Open ↗
3.3    (p → (q → r))   and   ((p → q) → r)
Open ↗

4. Validity

An argument is valid if there is no assignment making all premises true and the conclusion false. Premises are listed first; the conclusion is the last formula. Use the Check card and click Evaluate. If the argument is invalid, a counterexample row will be highlighted.
